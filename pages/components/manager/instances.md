# 实例管理

实例管理将分散在各服务器节点上的容器**集中呈现**，并提供管理员授权后的远程操作能力。同时承载控制台总览与联赛运营这两块"以实例为核心"的运营场景。

## 控制台总览

管理终端默认页，聚合以下数据源展示整体状态：

| 数据维度 | 数据源 |
| --- | --- |
| 节点离线数、迁移失败、TPS 异常实例 | 健康检查 + PubSub |
| 节点状态（PeerID、角色、CPU/内存、实例承载数、信誉分） | DHT + 共识层 |
| 实例总数、按状态聚合分布 | 共识层快照 |
| 节点间 Raft 连接、中继关系 | libp2p 周期上报 |

## 实例操作

所有实例操作均遵循"推送授权"流程：管理终端发起请求 → 服务端推送 AuthChallenge → 管理员 TEE 签名 → 服务端校验 → 执行并写入共识日志。详细授权流程见 [推送授权流程](./identity.md#推送授权流程)。

### 停止

管理员发起停止请求 → 系统反馈影响范围 → 管理员输入理由并确认 → 宿主节点收到 `StopRequest` → 完成 challenge/response 签名验证 → 优雅停止 MC 实例。

权限要求：发起者必须持有"管理员"或更高 VC。

### 迁移

分两种触发方式：

- **手动迁移**：管理员选择目标节点，调度器预先查询满足 placement 约束的可用节点供选择
- **自动迁移**：管理员只确认动作，目标节点由共识层调度器决策

迁移阶段：`pre-dump → S3 同步 → 目标启动 → 流量切换 → 完成`。任何阶段失败可触发回滚。

### 配置修改

仅适用于服务类实例（`service` 类型），`room` 类型不允许修改。

管理员提交新配置 → 展示与当前配置的差异 → 签名授权 → 按字段特性选择**热重载**或**重启生效**。重启会通知所有在线玩家，默认延迟 60 秒。

### 控制台日志

实例的实时日志流供技术管理员排查问题，支持跟随尾部、暂停、关键词搜索、导出（仅高级管理员，操作被审计）。

**控制台不接受输入**——所有写操作必须走显式的授权流程，禁止 RCON 后门。

## 批量操作

| 约束 | 说明 |
| --- | --- |
| 上限 | 一次最多 20 个实例 |
| 签名 | 同一动作只签名一次，签名内容包含批次 hash |
| 执行 | **串行**执行，允许中途中止 |
| 审计 | 批量操作有专用审计条目类型 |
| 权限 | 同一批次必须属于同一权限级别 |

## 联赛运营

[联赛系统](../../design/tournament.md) 定义了赛事/比赛/队伍的数据模型与流程。

### 赛事创建

赛事配置拆解为五个阶段：基本信息 → 赛制+地图 → 积分规则 → 报名条件 → 预览提交。大型赛事（参赛上限 > 64 或奖励池超过阈值）自动转为多签提案。

### 比赛运营

赛事开始后可管理进行中的 Match，包括实时状态监控、异常告警（玩家掉线、TPS 阈值、争议举报）、人工介入（暂停/重置/判定胜负）。

**判定胜负**是争议处理的兜底手段，需要至少两名管理员签名并进入治理流程。

### 争议处理

玩家提交的比赛争议包含：争议编号、提交人、类型（作弊/结算/行为）、证据、状态（open/investigating/resolved/rejected）。处理结论需签名，杜绝私自结案。

### 赛季管理

赛季数据由共识层维护，管理终端做展示与操作：

- 赛季概览：赛事汇总、排行榜
- 赛季归档：排行榜快照与 Match 结果导出
- 新赛季准备：复制模板、调整规则后发布

赛季切换需要全体核心成员多签。
